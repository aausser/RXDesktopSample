Ranorex Studio Desktop Testing Sample Solution

v.2.0, October 2019

Introduction

This sample solution represents a database test based on the Ranorex demo application which is included within the solution. Sufficient inline documentation guarantees self-explanation. Refer to our online user guide at www.ranorex.com/help/latest/ for further support and help.

The solution is divided into two test suites:

SmokeTest (if you fail - fail fast...)

The concept of smoke tests (i.e. preliminary testing) reveals simple failures severe enough to, for example recect a prospective software release. Smoke tests cover the most important functionality of a component or system. The current smoke test includes:
  1. StartCloseTest
    2. This test validates if the application under test (Ranorex Demo Application) can be started and closed.
  2. AUTVersionTest
    3. This test validates if the installed application refers to a specific version.

DatabaseTest

The database test represents a 3-step test consisting of the below listed test cases:
  1. Test_empty_database
    2. This test validates if the initial, empty database is setup correctly. This is done by validating if the initial database counter is set to '0'.
  2. Test_one_entry_database
    3. Here, one complete database entry is inserted into the form fields and added to the database. The test then validates if the database counter correctly increases from '0' to '1'.
  3. Data-driven_database_test
    4. Finally, the third test case demonstrates data-driven testing. 8 entries are read from an external data source (CSV file) and inserted into the database. It is validated, that the database counter correctly shows the final number of entries.

Controlling the test execution

The test execution can be controlled in as follows:

Test sequence

If there are two or more test suites within a project, opening the drop-down list of the toolbar RUN button opens access to the configuration of the test sequence.



Here you can change the sequence in which test suites are executed and select their run configurations.

Run configuration

Run configurations are a smart way of including/excluding test cases from a test suite during a test execution.



  1. Select one run configuration from the list
  2. Click the RUN button to execute this configuration
